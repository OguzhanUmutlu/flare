# Configuration

The `flare.json` file sits at the root of your project and defines how Flare compiles your datapack. You can quickly
generate a basic configuration file by running `flare init` in your project directory.

## Properties

Below is a list of all available properties you can define in your `flare.json`.

| Property             | Type      | Default              | Description                                                                                                        |
|----------------------|-----------|----------------------|--------------------------------------------------------------------------------------------------------------------|
| `namespace`          | `string`  | `"flare"`            | The primary namespace used for your datapack's functions and resources.                                            |
| `pack_format`        | `integer` | `15`                 | The target datapack format version included in your `pack.mcmeta`.                                                 |
| `description`        | `string`  | `"A Flare datapack"` | The description of the datapack shown in-game.                                                                     |
| `build_dir`          | `string`  | `"dist"`             | The directory where the compiled datapack will be output. This can be a relative or absolute path.                 |
| `validation_level`   | `string`  | `"strict"`           | Sets the internal Minecraft command schema validation level (`strict`, `warning`, `none`).                         |
| `minecraft_version`  | `string`  | `"1.20.4"`           | The Minecraft version to use for schema validation.                                                                |
| `nbt_schema_missing` | `string`  | `"error"`            | Action to take when indexing an NBT path that doesn't exist in the attached schema (`error`, `warning`, `ignore`). |

---

## Examples

### Basic Configuration

This is the standard configuration generated by `flare init`:

```json
{
  "namespace": "my_datapack",
  "pack_format": 15,
  "description": "My custom Flare datapack",
  "build_dir": "dist"
}
```

### Advanced Configuration

If you are working on a larger project and want to bypass strict validation for faster compile times, or output your
files to a specific server directory, you can expand your configuration:

```json
{
  "namespace": "minigame_core",
  "pack_format": 26,
  "description": "Core mechanics for the server.",
  "build_dir": "build/datapack",
  "validation_level": "none",
  "minecraft_version": "1.20.4",
  "nbt_schema_missing": "warning"
}
```

---

## CLI Overrides

Any property defined in your `flare.json` can be temporarily overridden at compile time using command-line flags.

For example, running `flare main.py --out-dir=test_build` will override the `build_dir` defined in your JSON and output
the datapack directly to the `test_build` directory instead.

For a full list of override flags, see the [CLI Reference](https://www.google.com/search?q=./cli.md).
